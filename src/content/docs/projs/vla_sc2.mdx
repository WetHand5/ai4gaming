<h3>背景简介</h3>
>&emsp;&emsp;2016年，AlphaGo首次用深度神经网络探索出了在不可完全解的游戏中用AI进行决策，击败了当时世界上最厉害的几位围棋选手，并彻底颠覆了该领域的
“游戏规则”

>&emsp;&emsp;2019年，DeepMind先公开展示了 AlphaStar 对战职业选手的结果，随后又在官方天梯 Battle.net 上以受限条件匿名对战，
最终报告称其在三族上都达到了 Grandmaster 水平，并高于 99.8% 的活跃排位玩家。这标志着使用**深度学习、模仿学习、强化学习和多智能体自博弈**训练出来的游戏智能体系统，
可以在**完整、实时、非完全信息、超大动作空间**的 RTS 环境中达到顶尖人类水平

>&emsp;&emsp;笔者开始写这个项目的时候水平有限，还无法完整理解AlphaStar的工程艺术，但是希望通过这个小项目能够粗略地感受这个领域的工程之美

<h3>框架设计</h3>
&emsp;&emsp;那么我们要用什么框架玩这个游戏呢？如果是单纯用强化学习来做，我相信通过合理的奖励设置，系统能够把这个游戏玩得不错。但这并不是我们的目标。
为了让他成为一个好的player,抽象一点来说他至少需要具有**高层长时序多关系Planning**，**可解释的策略Reasoning**，**稳定交互Action**能力

&emsp;&emsp;从工程上来说，这个框架至少应该分为三层：
>&emsp;&emsp;**1.游戏接口与动作表示**：解决环境和模型的交互问题

>&emsp;&emsp;**2.策略网络本体**：如何从部分可观测、长时间序列、多单位的状态中提取信息决策，并输出稳定的交互动作

>&emsp;&emsp;**3.训练架构**：怎么把这个网络训练到真的“会玩”而不是成为只会一种套路不会变通的“背版”玩家

<h3>AlphaStar的设计</h3>
DeepMind在官方材料中把AlphaStar核心的网络概括成：


**基于units的Transformer Torso**
>&emsp;&emsp;游戏状态首先被整理成一组实体，每个实体是一个 unit token，带着结构化特征。Transformer Torso的职责不是做语言，而是做多单位关系建模。因为在 SC2 里，单个单位的意义往往取决于它和其他单位的关系：我方兵线密度、敌方站位、近战远程搭配、建筑布局、局部兵力对比。这种“集合里的元素之间相互影响”的问题，Transformer 比单纯 MLP 或简单卷积更适合。你以把它理解成：先把每个单位编码成向量，再让 self-attention 学“谁和谁相关”

**深层 LSTM core**
>&emsp;&emsp;只有 Transformer 还不够，因为 SC2 是带记忆的部分可观测游戏。你当前没看到敌人，不代表敌人不存在，只是信息暂时不可见。所以 AlphaStar 在 torso 之后接了一个深层 LSTM core。这个模块的职责是跨时间整合信息，把过去若干秒乃至更久之前的侦察、交战、经济发展、科技路径这些线索保存在隐状态里。

**自回归policy head**
>&emsp;&emsp;所谓自回归，不是一次性输出完整动作，而是把一个复杂动作拆成多个子决策顺序生成。比如可以先决定动作函数类型，再决定延迟、是否排队、选哪些单位、目标位置或目标实体等。这样做有两个好处：第一，能处理组合爆炸的动作空间；第二，能够显式表达动作参数之间的条件依赖关系。

**Pointer network**
>&emsp;&emsp;在 SC2 里，很多动作不是指向固定类别，而是要从“当前可见的一堆实体”中选一个或多个对象，例如选某个单位、指定某个目标单位、某个建筑、某个屏幕位置对应的对象。普通 softmax 头不擅长“从输入集合里指向其中一个元素”，pointer network 正好适合这个场景。所以 AlphaStar 把它用于实体选择类动作，本质上是在说：有些动作参数不是从固定词表采样，而是从当前输入对象集合中“指过去”。


**Centralilzed value baseline**
>&emsp;&emsp;除了 policy，AlphaStar 还有 centralized value baseline。这是训练期用来降低策略梯度方差、稳定强化学习的重要模块。之所以叫 centralized，是因为在多智能体/自博弈背景下，价值评估可以利用比单个行动者局部决策更丰富的训练信号。直观上，policy 负责“怎么出招”，value/baseline 负责“当前局面对长期胜负大概有多好”，从而给训练提供更平滑的优化目标。
